SECURITY.PY CONCEPTS EXPLAINED
================================================================================

A. Password Hashing with bcrypt
- Uses passlib's CryptContext to hash passwords securely.
- bcrypt is a slow hashing algorithm that prevents brute-force attacks.
- Never store plain text passwords - always hash them!

B. JWT (JSON Web Tokens)
- Stateless authentication method.
- Contains user data (like email) signed with a secret key.
- Expires after a set time (30 minutes in your app).
- Used for API authentication without sessions.

C. Authorization Header
- HTTP header that carries the JWT token.
- Format: "Authorization: Bearer <token>"
- Frontend sends this with every authenticated request.

D. FastAPI Dependencies
- Functions that run before your endpoint code.
- get_current_user_dependency automatically gets the logged-in user.
- Raises 401 error if token is invalid or missing.

E. Token Flow
- User logs in → Server creates JWT → Returns to frontend
- Frontend stores token → Sends in Authorization header
- Server decodes token → Gets user email → Fetches user from DB

F. Security Best Practices
- SECRET_KEY should be long and random (change in production).
- Tokens expire to prevent stolen tokens from being used forever.
- Use HTTPS in production to encrypt token transmission.
