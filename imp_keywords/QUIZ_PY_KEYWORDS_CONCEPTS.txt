================================================================================

QUIZ.PY CONCEPTS EXPLAINED
================================================================================

A. Quiz Management
- Create quizzes with title, subject, grade, description
- Store total_questions count
- Get all quizzes or specific quiz
- Delete quiz and all associated questions

B. Question Handling
- Add questions to quiz
- Each question has 4 options (a, b, c, d)
- Store correct_answer as letter (a, b, c, or d)
- Questions linked to quiz via quiz_id

C. Quiz Submission Process
1. User submits answers as dictionary
2. Dictionary format: {question_id: "a", question_id: "b", ...}
3. Compare user answers with correct answers
4. Count correct and wrong answers
5. Calculate percentage score

D. Scoring Calculation
- score = (correct / total * 100)
- Example: 5 correct out of 10 = 50%
- Store as Float percentage (72.5, 85.0, etc)

E. Answer Recording
- Save each user answer to UserAnswer table
- Store: user_id, quiz_id, question_id, user_answer, is_correct
- Creates detailed record of what user answered

F. Progress Recording
- Save overall quiz result to Progress table
- Store: user_id, quiz_id, total_questions, correct_answers, wrong_answers, score
- User can see past quiz results

G. Case-Insensitive Comparison
- .lower() converts to lowercase before comparing
- "A" == "a" after .lower()
- Prevents incorrect marking due to case

H. Database Count
- func.count(Question.id) counts total questions
- Quiz.total_questions field gets updated
- Useful for quizzes with dynamic question count

I. Dictionary in Request
- QuizSubmission.answers is a dict
- Format: {question_id_str: answer_letter}
- Example: {"1": "a", "2": "b", "3": "c"}

J. Cascade Delete
- Delete questions before deleting quiz
- Without this, foreign key errors occur
- Maintains referential integrity
