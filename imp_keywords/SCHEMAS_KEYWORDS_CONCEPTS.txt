SCHEMAS/ CONCEPTS EXPLAINED
================================================================================

A. What are Schemas?
- Data validation and serialization classes
- Define what data frontend sends (request) and backend returns (response)
- Prevent invalid data from entering your API

B. Pydantic BaseModel
- Automatic type validation
- Auto-generates JSON Schema for API docs
- Converts data types automatically (e.g., "123" → 123)

C. Base vs Create vs Response
- Base: Common fields shared by multiple schemas
- Create: What frontend sends when creating (no id, no timestamps)
- Response: What backend sends back (includes id, timestamps)
- This prevents exposing unwanted fields like hashed passwords

D. EmailStr Validation
- Automatically validates email format
- Rejects invalid emails before reaching endpoint
- Example: "user@example.com" ✅ | "notanemail" ❌

E. Optional and Default Values
- Optional[str] = None means field can be null
- color: str = "#fff7b1" means default color is yellow
- Useful for optional request parameters

F. from_attributes = True
- Tells Pydantic to read data from ORM models
- Converts SQLAlchemy User object → JSON response
- Works with db.query() results

G. Schema Hierarchy
- Parent schemas share common fields
- Child schemas add specific fields
- Example: QuizWithQuestions extends QuizResponse + adds questions list

H. Request vs Response
- Request: What user sends (CreateSchema)
- Response: What API returns (ResponseSchema)
- Keeps frontend safe from internal fields
