================================================================================

AUTH.PY CONCEPTS EXPLAINED
================================================================================

A. APIRouter
- Groups related endpoints together
- prefix="/api/auth" adds to all routes
- tags=["Authentication"] for documentation

B. Signup Process
1. Check if email already registered
2. Hash password with bcrypt
3. Create User object
4. Save to database
5. Return user data (without password!)

C. Login Process
1. Get email and password from form
2. Find user by email
3. Verify password matches hashed version
4. Create JWT token with email
5. Return token and token_type

D. OAuth2PasswordRequestForm
- Standard HTML form submission
- form_data.username = email
- form_data.password = password
- Browser sends as form data, not JSON

E. Authentication Flow
- Login endpoint generates token
- Frontend stores token
- GET /me endpoint verifies token
- Token sent in Authorization header

F. Password Security
- Never store plain text passwords
- Always hash before saving
- Use verify to compare on login
- Update password by hashing new password

G. Database Integrity
- Check email doesn't already exist before signup
- Verify password before changing it
- Use db.refresh() to get updated data
- Use db.delete() then db.commit() to remove

H. Error Handling
- 400: Email already registered (bad request)
- 401: Invalid password (unauthorized)
- 404: User not found (not found)
- Return clear error messages

I. Profile Updates
- Only update fields that are provided
- Optional fields allow partial updates
- Use db.refresh() to return updated data

J. Account Deletion
- Removes user completely from database
- All related data deleted (depends on cascade settings)
- Return success message
